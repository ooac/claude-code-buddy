## 📝 关键决策
- 插件形态：CLI（本地执行）
- 切换策略：支持随机 + 指定目标双模式
- 概率机制：官方风格默认值 + 可扩展
- 写配置策略：自动写入并自动备份
- 情绪风格：热血抽卡风

## 🚨 风险与问题
- 已知限制：若存在 `oauthAccount.accountUuid`，`/buddy` 可能优先读取 accountUuid，单改 userID 可能不生效（已通过 `doctor` 给出诊断提示）
- 已修复：`companion` 与种子推演错配问题（切换时已自动同步并在 `card` 中增加一致性诊断）
- 已修复：哈希分支错位（FNV 与 Bun.wyhash 不一致）导致的物种/精灵错位
- 已修复：连续抽卡交互与名称错配感（回车继续抽卡，名字随物种刷新）
- 已修复：运行态热更新滞后引发的“名称变了但图未变”误判（仅告警，不自动重启）
- 已修复：跨平台分发不可迁移与 Windows 控制台乱码风险（增加 portable 包与 UTF-8/ASCII 双模式）
- 已实现：手动宠物备份池，可按备份 ID 恢复任意历史宠物
- 已修复：一键界面未暴露备份入口（新增 `b/c` 交互快捷操作）
- 已修复：备份列表快捷键易误读与恢复路径分叉（改为 `c` 子界面内查看并按 `1~5` 恢复，主模式移除 `r`）
- 已修复：一键启动自动抽卡导致“刚打开就切宠物”（改为默认待机，回车才抽卡）
- 已优化：备份子界面序号可视化（`[1]~[5]` 数字黄色高亮）
